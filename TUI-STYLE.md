# Perihelion TUI 风格指南

> 版本：1.1 · 2026-03-30
> 适用范围：`rust-agent-tui` ratatui 界面

---

## 设计哲学

**极简锋利，单色制胜。**

终端界面的本质是信息密度与操作效率的平衡。这套风格拒绝装饰性颜色，用接近黑色的暖底色承载所有内容，用唯一的高饱和强调色点燃关键状态——其他一切靠亮度和对齐来建立秩序。

> "能用空白解决的，不用线条。能用亮度解决的，不用颜色。"

---

## 关键词

`极简` · `锋利` · `暖黑` · `单色强调` · `仪表盘感`

## 色彩系统

终端 TUI 使用 `Color::Rgb(r, g, b)` 精确控制，全部通过 `src/ui/theme.rs` 统一管理。

### 背景层（透明优先）

| 用途 | Color::Rgb | 说明 |
|------|-----------|------|
| 全局背景 | 透明（terminal 决定） | 不设置 bg，让终端底色透出 |
| 弹窗/面板底色 | `Rgb(10, 8, 6)` | `#0A0806`，用于 Clear 覆盖后的 popup |
| 弹窗表面 | `Rgb(18, 14, 10)` | `#120E0A`，popup 内部区块 |
| 选中行背景 | `Rgb(38, 22, 10)` | `#261608`，光标所在行，不用高亮反白 |

### 文字层（三级亮度）

| 用途 | Color::Rgb | ratatui 常量名 | 说明 |
|------|-----------|----------------|------|
| 主要文字 | `Rgb(218, 206, 208)` | `TEXT` | `#DACED0`，冷白偏暖，主体内容 |
| 次要文字 | `Rgb(140, 125, 120)` | `MUTED` | `#8C7D78`，标签、路径、辅助 |
| 禁用/极弱 | `Rgb(72, 62, 58)` | `DIM` | `#483E3A`，占位、已完成项 |

### 语义色（功能颜色，精准不泛滥）

| 用途 | Color::Rgb | ratatui 常量名 | 说明 |
|------|-----------|----------------|------|
| **强调（单一主色）** | `Rgb(255, 107, 43)` | `ACCENT` | `#FF6B2B`，边框激活/光标/关键操作 |
| 成功 / SAGE | `Rgb(110, 181, 106)` | `SAGE` | `#6EB56A`，工具成功、确认状态 |
| 警告 | `Rgb(176, 152, 120)` | `WARNING` | `#B09878`，次要强调（标题/工具名/快捷键） |
| 错误 | `Rgb(204, 70, 62)` | `ERROR` | `#CC463E`，拒绝/报错 |
| 思考（CoT） | `Rgb(167, 139, 250)` | `THINKING` | `#A78BFA`，推理过程，哑光紫 |
| ~~工具名~~ | ~~= SAGE~~ | `TOOL_NAME` | **已废弃**，按危险度分级（见"工具颜色映射"节） |
| 模型信息 | `Rgb(160, 130, 95)` | `MODEL_INFO` | `#A0825F`，状态栏模型名，棕金 |

### 边框（最小化原则）

| 用途 | Color::Rgb | 说明 |
|------|-----------|------|
| 空闲边框 | `Rgb(48, 38, 32)` | `#302620`，极低对比，只做功能性分隔 |
| 主输入框激活边框 | = ACCENT (`#FF6B2B`) | 命令输入框激活态，橙色仅保留于此 |
| 配置面板激活边框 | `Rgb(140, 125, 120)` | `MUTED`，ModelPanel/AgentsPanel 等次要交互区 |
| HITL/AskUser 弹窗边框 | = WARNING (`#B09878`) | 警示感，不刺眼 |
| 子 agent 组 | = SAGE | 子 agent 执行组边框，语义化 |

## 信息层次规则

**只用亮度，不堆颜色。** 同一区块内最多出现 2 种语义色（主色 + 1 功能色）。

```
第一层 TEXT      #DACED0  ████  —— 需要立即看到的：消息内容、选中项名称
第二层 MUTED     #8C7D78  ███   —— 需要参考的：时间戳、路径、标签、快捷键说明
第三层 DIM       #483E3A  ██    —— 可忽略的：分隔符、已完成 TODO、占位文字
语义色 ACCENT    #FF6B2B  ●     —— 仅用于：命令输入框激活边框、bash 工具名
语义色 WARNING   #B09878  ●     —— 用于：H1/H2 标题、写操作工具名、HITL 弹窗、快捷键
```

**原则**：
- 不用背景色标注文字（除光标行 `CURSOR_BG`）
- 颜色不用于分类，只用于状态（活跃/成功/警告/错误）
- Bold = 比普通文字重要一级，等价于半个颜色层级

## 边框与线条

TUI 中线条极容易产生「框框套框框」的视觉噪音。

**规则**：
- 主界面（消息列表、输入框）：单线边框 `Borders::ALL`，空闲色 `#302620`
- 激活态（输入框 focus）：边框色切换到 `ACCENT`，不加粗线型
- 弹窗：必须先 `render_widget(Clear, area)`，再画边框，避免透底
- 区块分隔：优先用空行，其次用 `─` 字符，最后才考虑 `Borders`
- 状态栏：**无边框**，靠底部位置天然分隔

**线型约定**：
```
普通分隔  ─────────────────
标题分隔  ── 标题内容 ──────
列表项    ▶ 当前项（ACCENT）
          ○ 普通项
          ● 选中项
```

## 布局与弹窗

### 主布局（三段式）

```
┌─────────────────────────────────────────────┐
│ 标题栏（单行）                                │  ← 模型名 / Agent 名 / 状态
├─────────────────────────────────────────────┤
│                                             │
│              消息列表（主区域）               │  ← 可滚动，占大部分高度
│                                             │
├─────────────────────────────────────────────┤
│ 输入框（2-4行）                              │  ← TextArea
├─────────────────────────────────────────────┤
│ 状态栏（单行）                                │  ← 快捷键 / TODO / 进度
└─────────────────────────────────────────────┘
```

### 底部展开区（替代模态弹窗）

**所有交互型面板（HITL、AskUser、模型选择、Agent 选择、Relay、Thread 浏览）** 均从底部展开，不遮挡消息历史。

```
┌─────────────────────────────────────────────┐
│              消息列表（压缩）                 │
├─────────────────────────────────────────────┤
│ ╔═══════════════════════════════════════╗   │  ← 底部展开区
│ ║  HITL: 确认执行 bash?                ║   │     ACCENT 边框标记激活
│ ║  > rm -rf node_modules               ║   │
│ ║  [y] 确认  [e] 编辑  [r] 拒绝        ║   │
│ ╚═══════════════════════════════════════╝   │
├─────────────────────────────────────────────┤
│ 状态栏                                       │
└─────────────────────────────────────────────┘
```

实现策略：
- 计算展开区高度（内容自适应，最大 40% 屏高）
- 消息列表区域相应压缩，scroll_offset 不重置
- 展开区边框用 `ACCENT` 颜色标记当前模式
- Esc 收起，恢复原布局

### 已废弃：居中模态弹窗
原有的 `render_widget(Clear, popup_area)` 居中弹窗模式**逐步迁移**为底部展开区。迁移优先级：HITL → AskUser → 模型面板 → 其他。

## 状态栏设计

状态栏始终可见，承载当前上下文的所有关键信息。

**布局（从左到右）**：
```
[模型名] [Agent名?]  ─────  [任务时长]  [快捷键提示]  [加载指示]
```

**颜色编码**：
- 模型名：`MODEL_INFO` 棕金色，不抢眼
- Agent 名：`MUTED` 灰色（v1.1 降噪）
- 任务时长：`MUTED` 灰色；loading 时为 `LOADING` 电光青色（v1.1 降噪）
- 快捷键说明：快捷键本身用 `WARNING` 黄色加粗，说明文字用 `MUTED`
- 错误状态：`ERROR` 红色

**Badge 式指示（不用底色块）**：
```
loading:  ⠋ 处理中  （WARNING 色 spinner 字符）
done:     ✓ 0.8s   （SAGE 色）
error:    ✗ 出错   （ERROR 色）
```

## 消息渲染

### 消息类型颜色

| 类型 | 颜色规则 |
|------|---------|
| 用户消息 | 序号 ACCENT，内容 TEXT |
| AI 消息 | 序号 ACCENT，内容 TEXT |
| System 消息 | 全部 MUTED |
| 工具调用 | 工具名 `TOOL_NAME`（=SAGE），参数 MUTED |
| 工具结果 | 内容 TEXT，错误时整体 ERROR |
| 思考内容 | `THINKING` 紫色 italic，前缀 `│` |
| 子 Agent 组 | 边框用 SAGE，标题 SAGE + Bold |

### Markdown 渲染

| 元素 | 样式 |
|------|------|
| H1/H2 | ACCENT + Bold + 下划线 |
| H3 | WARNING + Bold |
| 代码块 | 前缀 `│` SAGE，内容 TEXT（无底色） |
| 行内代码 | ACCENT 色，无底色 |
| 链接 | SAGE + 下划线 |
| 加粗 | Bold（不换颜色） |
| 引用 | MUTED，前缀 `▎` |

## 工具颜色映射

按操作危险度三级分级，颜色不滥用：

| 工具 | 颜色 | 理由 |
|------|------|------|
| `bash` | ACCENT（橙） | 最高权限，最需要注意 |
| `write_file` / `edit_file` / `folder_operations` / `rm_*` | WARNING（暖米灰） | 破坏性/结构性操作，警示但不刺眼 |
| `read_file` / `glob_files` / `search_files_rg` / `launch_agent` / `ask_user_question` / `todo_write` | MUTED（灰） | 只读/委派/安全操作，无需颜色强调 |
| 工具**执行成功**（结果区，非工具名） | SAGE（绿） | 成功状态，与工具名颜色解耦 |
| 工具**执行出错** | ERROR（红） | 任何工具出错 |

## ratatui theme.rs 目标配置

```rust
use ratatui::style::Color;

// ── 强调色（单一主色）────────────────────────────────────
pub const ACCENT: Color      = Color::Rgb(255, 107, 43);   // #FF6B2B 橙红

// ── 功能色 ───────────────────────────────────────────────
pub const SAGE: Color        = Color::Rgb(110, 181, 106);  // #6EB56A 哑绿
pub const WARNING: Color     = Color::Rgb(176, 152, 120); // #B09878 暖米灰
pub const ERROR: Color       = Color::Rgb(204, 70, 62);    // #CC463E 暗红
pub const THINKING: Color    = Color::Rgb(140, 110, 180);  // #8C6EB4 哑紫

// ── 文字层级 ─────────────────────────────────────────────
pub const TEXT: Color        = Color::Rgb(218, 206, 208);  // #DACED0 主文字
pub const MUTED: Color       = Color::Rgb(140, 125, 120);  // #8C7D78 次要
pub const DIM: Color         = Color::Rgb(72, 62, 58);     // #483E3A 极弱

// ── 特殊 ─────────────────────────────────────────────────
pub const TOOL_NAME: Color   = SAGE;
pub const SUB_AGENT: Color   = SAGE;
pub const MODEL_INFO: Color  = Color::Rgb(160, 130, 95);   // #A0825F 棕金
pub const BORDER: Color      = Color::Rgb(48, 38, 32);     // #302620 空闲边框
pub const BORDER_ACTIVE: Color = ACCENT;
pub const CURSOR_BG: Color   = Color::Rgb(38, 22, 10);     // #261608 光标行底色
```

## 刻意回避

- ❌ `Color::Cyan` / `Color::Blue` / `Color::Green` 直接使用（用 Rgb 精确控制）
- ❌ 背景色标注文字（除 `CURSOR_BG` 光标行）
- ❌ 多种装饰色同时出现在同一视图（最多 2 种语义色）
- ❌ 双线边框 `╔══╗`（除非明确标记「这是需要操作的区域」）
- ❌ 居中模态弹窗遮挡消息历史
- ❌ Bold + 颜色 + 下划线三重强调（选一种）
- ❌ 渐变/彩虹色工具标记（工具颜色限定在以上色板内）

## 配色变更说明（v1.0 → v1.1）

**问题**：ACCENT 橙色调过度使用，视觉重心不明确。

**改动原则**：降噪优先，橙歇山顶——橙色只留给最高优先级交互。

| 区域 | v1.0 | v1.1 |
|------|------|------|
| H1/H2 标题 | ACCENT + BOLD + 下划线 | WARNING + BOLD（去下划线） |
| 工具名 | 全 SAGE | bash=ACCENT / 写操作=WARNING / 只读=MUTED |
| 工具执行成功 | TEXT | SAGE（语义收窄） |
| 配置面板边框 | ACCENT | MUTED |
| HITL/AskUser 弹窗边框 | ACCENT | WARNING |
| 状态栏时间/Agent 名 | ACCENT | MUTED |
| THINKING 色值 | #8C6EB4 | #A78BFA（与 theme.rs 一致） |
